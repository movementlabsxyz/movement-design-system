The Movement Design System is now production-ready with a stable API, comprehensive component library, and full TypeScript support. This release marks the completion of the migration to shadcn/ui architecture with Tailwind CSS v4.

### What's New in 1.0

- **50+ Production-Ready Components** - Complete set of accessible, customizable React components
- **Stable API** - All components follow consistent shadcn/ui patterns  
- **Full TypeScript Support** - Complete type definitions for all components
- **Comprehensive Documentation** - Interactive Storybook with live examples
- **Tailwind CSS v4** - Modern CSS-first configuration
- **Tree-Shakeable** - Optimized bundle with proper exports
- **Dark Mode Ready** - Built-in dark mode support for all components

### Changed

#### Styling Recipes
- **BREAKING**: Moved styling recipes to dedicated `recipes.css` file
- **BREAKING**: Removed `gradientBorderStyles` CSS string export - styles now automatically included
- **BREAKING**: Removed `glassBackgroundStyles` CSS string export - styles now automatically included  
- **BREAKING**: Removed `gradientBackgroundStyles` CSS string export - styles now automatically included
- Improved developer experience: no manual style injection required
- Class name exports remain unchanged: `gradientBorderClasses`, `glassBackgroundClasses`, `gradientBackgroundClasses`
- Updated all components to remove inline style injections
- Updated documentation to reflect automatic CSS inclusion

**Migration Guide:**
- Remove any manual `<style dangerouslySetInnerHTML={{ __html: gradientBorderStyles }} />` injections
- Simply import the design system CSS: `import "@movementlabsxyz/movement-design-system/component-styles"`
- Continue using class name exports as before: `gradientBorderClasses.glow`, etc.
