# Movement Design System

**Version 1.0** - Production-ready, stable API, comprehensive documentation.

A comprehensive design system built with shadcn/ui components for Movement Labs applications. This library provides a collection of beautifully designed, accessible, and customizable React components powered by Tailwind CSS v4.

## üöÄ Installation

### Prerequisites

- React 18+ or 19+
- Tailwind CSS v4
- PostCSS

### Step 1: Install the Design System

```bash
npm install @movementlabsxyz/movement-design-system
# or
pnpm add @movementlabsxyz/movement-design-system
# or
yarn add @movementlabsxyz/movement-design-system
```

### Step 2: Set Up Tailwind CSS

Ensure you have a standard Tailwind CSS setup with PostCSS. If you haven't set this up yet, follow the [Tailwind CSS v4 installation guide](https://tailwindcss.com/docs/installation).

### Step 3: Import Styles

Add the following imports to your `global.css` file **in this exact order** (the order is important!):

```css
@import "movement-design-system/component-styles";
@import "movement-design-system/theme";
@import "tailwindcss";
```

> ‚ö†Ô∏è **Important**: The order of these imports matters! Make sure to import the component styles and theme before Tailwind CSS.

## üß© Blocks - Pre-built Patterns

In addition to individual components, the design system includes **Blocks**: pre-built, production-ready patterns composed from multiple components. These blocks serve as examples and starting points for common UI patterns in Movement applications.

### What are Blocks?

Blocks are complete, copy-paste-ready code examples that demonstrate how to combine components from the design system to create real-world UI patterns. Each block is fully functional and can be customized to fit your specific needs.

### Available Blocks

- **StakingBalance** - Glass-morphism card displaying staking stats, total staked amount, active delegations, and rewards
- **StakeForm** - Complete staking form with validator selection, amount input, and gradient styling
- **ValidatorsTable** - Comprehensive validators table with sorting, progress bars, and detailed metrics
- **MobileValidatorsTable** - Responsive validators table that automatically switches between table and card layouts
- **PaginatedTable** - Full-featured table with sorting, pagination, and multiple style variants

### Using Blocks

Browse the **Blocks** section in the sidebar to see live examples with full source code. Simply copy the code and adapt it to your needs‚Äîall blocks are built exclusively with components from this design system.

## ‚öôÔ∏è Recommended Setup

### Automatic Class Sorting with Prettier

We recommend using Prettier's automatic class sorting for Tailwind CSS to keep your classes organized:

1. Install the Prettier plugin:

```bash
npm install -D prettier prettier-plugin-tailwindcss
# or
pnpm add -D prettier prettier-plugin-tailwindcss
```

2. Add to your `.prettierrc` or `prettier.config.js`:

```json
{
  "plugins": ["prettier-plugin-tailwindcss"]
}
```

For more information, see the [official Tailwind CSS blog post](https://tailwindcss.com/blog/automatic-class-sorting-with-prettier).
